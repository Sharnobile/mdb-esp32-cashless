## Goal
Create a Reports page with date range selection, sales data preview table, and two CSV export formats: DATEV Buchungsstapel (for German accountants/tax advisors) and a simple CSV with all requested columns.

## Purpose
Final phase of the Steuer-Berichte milestone. Users need to export tax-compliant sales data for their accounting software (DATEV) or for manual analysis (simple CSV). This completes the workflow: configure taxes → backfill → export.

## Output
- New page: `/reports` with date range picker, preview table, and 2 export buttons
- New composable: `useReports` for data fetching and CSV generation
- Nav integration: Reports link in sidebar Operations group
- Full i18n (EN + DE)
</objective>

<task type="auto">
  <name>Task 1: Create useReports composable with data fetching and CSV generation</name>
  <files>management-frontend/app/composables/useReports.ts</files>
  <action>
    Create composable exposing:

    **Types:**
    ```typescript
    interface ReportSale {
      id: string
      created_at: string
      machine_name: string
      machine_id: string
      item_number: number
      product_name: string | null
      category_name: string | null
      item_price: number
      tax_rate_snapshot: number | null
      tax_amount: number | null
      price_net: number | null
      channel: string | null
    }
    ```

    **State + Functions:**
    - `sales: Ref<ReportSale[]>` — fetched report data
    - `loading: Ref<boolean>`
    - `dateFrom: Ref<string>` — default: first of current month (YYYY-MM-DD)
    - `dateTo: Ref<string>` — default: today (YYYY-MM-DD)
    - `fetchReportData()` — query sales joined with vendingMachine, machine_trays→products→product_category for the date range, filtered by company (RLS handles this)
    - `exportSimpleCsv()` — generates UTF-8 BOM CSV with German locale formatting (comma decimal, semicolon separator), triggers download
    - `exportDatev()` — generates DATEV EXTF Buchungsstapel CSV (Windows-1252, semicolon, 2 header rows + data)

    **Simple CSV columns:**
    Datum;Uhrzeit;Automat;Automaten-ID;Tray;Produkt;Kategorie;Brutto (EUR);MwSt. (%);MwSt. (EUR);Netto (EUR);Bezahlart

    **DATEV EXTF format:**
    Row 1 (metadata): "EXTF";700;21;"Buchungsstapel";...  (semicolon-separated, key fields: format type 21, version 700, date range, SKR03)
    Row 2 (headers): "Umsatz (ohne Soll/Haben-Kz)";"Soll/Haben-Kennzeichen";"WKZ Umsatz";"Konto";"Gegenkonto (ohne BU-Schlüssel)";"BU-Schlüssel";"Belegdatum";"Belegfeld 1";"Buchungstext";... (127 columns total, most empty)
    Data rows:
    - Umsatz: gross amount (German format: "2,80")
    - Soll/Haben: "S" (Soll/debit for revenue)
    - WKZ: "EUR"
    - Konto: "10000" (Kasse/Bank — configurable later)
    - Gegenkonto: "8400" (Erlöse 19% SKR03) or "8300" (Erlöse 7% SKR03)
    - BU-Schlüssel: "9" for 19%, "8" for 7% (standard DATEV tax keys), empty if unknown
    - Belegdatum: DDMM format (e.g. "0504" for April 5)
    - Belegfeld 1: sale ID (truncated to 12 chars)
    - Buchungstext: "Automat {name} Tray {nr}" (max 60 chars)

    **Encoding helper:**
    For DATEV: convert string to Windows-1252 using TextEncoder fallback (replace non-1252 chars with '?')
    For simple CSV: prepend UTF-8 BOM (\uFEFF)

    **Download helper:**
    Create Blob, create object URL, trigger download via temporary `<a>` element

    Avoid:
    - Fetching all sales at once for huge date ranges (use pagination or limit + warning)
    - Breaking if tax columns are NULL (show empty in CSV, skip BU-Schlüssel in DATEV)
  </action>

    - TypeScript compiles without errors
