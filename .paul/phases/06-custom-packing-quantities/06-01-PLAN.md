<objective>
## Goal
Add adjustable quantities in the packing step of the refill wizard so operators can control how many of each product to pack per machine, instead of the current all-or-nothing checkbox behavior.

## Purpose
When warehouse stock is limited, operators need to distribute products across multiple machines rather than committing all available stock to the first machine. Currently, checking an item commits `min(deficit, available)` — the operator has no control. This feature lets them reduce quantities per machine to spread stock evenly and test product performance before committing full stock.

## Output
Modified composable and UI: packing checklist items show a compact quantity adjuster when checked, defaulting to the effective deficit but allowing reduction. Combined mode quantities reflect per-machine custom settings.
</objective>

  <action>
    1. Add `customQuantities: ref(new Map<string, Map<string, number>>())` state
       - Key: machineId → Map<productId, quantity>
       - Represents the user-chosen quantity for each checked item

    2. Modify `togglePacked()`:
       - When checking an item (adding to set): auto-set customQuantities for that machine+product to `effectiveDeficit(item, machineId)` (the default max)
       - When unchecking: remove from customQuantities

    3. Modify `recalculateCommittedQuantities()`:
       - Instead of `Math.min(item.deficit, avail)`, use `Math.min(customQty, item.deficit, avail)` where customQty comes from customQuantities map
       - If no custom qty set, fall back to current behavior (full deficit)

    4. Add `setCustomQuantity(machineId: string, productId: string, qty: number)`:
       - Clamp qty between 1 and min(deficit, total warehouse stock for product)
       - Update customQuantities map
       - Call recalculateCommittedQuantities()

    5. Add `getCustomQuantity(machineId: string, productId: string): number | null`:
       - Returns the custom qty or null if not set

    6. Modify `togglePackedCombined()`:
       - When unchecking combined: clear customQuantities for that product across all machines
       - When checking combined: set default quantities per machine (existing effectiveDeficit)

    7. Update `effectiveDeficitCombined()`:
       - Sum custom quantities (when checked) instead of raw committed quantities

    8. Add `resetWizard()`: clear customQuantities (already clears other state)

    9. Export `setCustomQuantity`, `getCustomQuantity`, `customQuantities` from the composable

    Avoid:
    - Do NOT persist customQuantities to localStorage — they only matter during packing step
    - Do NOT change the PersistedTourState interface
    - Do NOT modify startTour() — it already reads from committedQuantities which will reflect custom amounts
