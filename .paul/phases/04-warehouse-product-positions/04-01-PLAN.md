<objective>
## Goal
Add a warehouse product positioning system so operators can define the physical order of products in each warehouse. This is the foundation for sorted pick lists in the refill tour.

## Purpose
Operators currently waste time zigzagging through the warehouse during refill packing. By storing a `sort_order` per product per warehouse, the refill wizard (Phase 05) can present pick lists matching the physical warehouse layout — enabling a single linear walk from start to end.

## Output
- Migration with `warehouse_product_positions` table + RLS policies
- `useWarehouse` composable extended with position CRUD + ordered product list
- "Lagerplätze" (Positions) tab in `/warehouse` page for drag-and-drop reordering
</objective>

## AC-1: Product positions stored per warehouse
```gherkin
Given a warehouse with products in stock
When an admin assigns sort_order values to products
Then the positions are persisted in warehouse_product_positions
And each product has exactly one position per warehouse (unique constraint)
```

## AC-2: RLS enforces company isolation
```gherkin
Given two companies each with their own warehouses
When company A queries warehouse_product_positions
Then only positions for company A's warehouses are returned
```

## AC-3: Position management UI
```gherkin
Given an admin viewing the warehouse page
When they navigate to the "Positionen" tab
Then they see all stocked products listed in sort_order
And they can drag to reorder (or use move up/down buttons)
And the new order is saved automatically
```

## AC-4: Products without positions sort last
```gherkin
Given a warehouse with some positioned and some unpositioned products
When fetching the ordered product list
Then positioned products appear first (by sort_order ASC)
And unpositioned products appear after (alphabetically by name)
```

    1. `warehouse_product_positions` table:
       - `id` uuid PK (gen_random_uuid)
       - `warehouse_id` uuid NOT NULL FK → warehouses(id) ON DELETE CASCADE
       - `product_id` uuid NOT NULL FK → products(id) ON DELETE CASCADE

    2. RLS policies (same pattern as warehouse_stock_batches):
       - SELECT: warehouse_id IN (SELECT id FROM warehouses WHERE company_id = my_company_id())
       - INSERT/UPDATE/DELETE: same warehouse check AND i_am_admin()

    3. Index on (warehouse_id, sort_order) for efficient ordered queries
