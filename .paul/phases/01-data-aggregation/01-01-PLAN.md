<objective>
## Goal
Create a PostgreSQL RPC function `get_machine_insights_kpis(p_machine_id uuid, p_company_id uuid, p_days int)` via a new migration that returns pre-aggregated KPI data for a single vending machine, structured for direct consumption by an AI prompt.

## Purpose
The AI insights edge function (Phase 02) needs structured KPI data, not raw rows. Pre-aggregating in SQL keeps edge function code simple, reduces token usage in Claude prompts, enforces RLS via `p_company_id` validation, and keeps logic close to the data. This is the foundation the entire AI feature builds on.

  <name>Task 1: Create migration with get_machine_insights_kpis RPC</name>
  <files>Docker/supabase/migrations/20260317000000_machine_insights_rpc.sql</files>
  <action>
    Create a new migration file that defines a PostgreSQL function with this exact signature:

    ```sql
    create or replace function get_machine_insights_kpis(
      p_machine_id uuid,
      p_company_id uuid,
      p_days int default 30
    )
    returns json
    language plpgsql
    security definer
    as $$
    ```

    The function must:

    1. **Validate company ownership**: Check that a `vendingMachine` row with `id = p_machine_id` AND joined `embeddeds.company_id = p_company_id` exists. If not found, return null immediately.

    2. **Compute the time window**: `now() - (p_days || ' days')::interval`

    3. **Fetch tray-level data** by joining `machine_trays` → `products` → `sales` (LEFT JOIN sales on `sales.item_number = machine_trays.item_number AND sales.machine_id = p_machine_id AND sales.created_at >= window_start`):
       - `item_number`, `product_name` (products.name, or 'Empty Slot' if product_id is null)
       - `capacity`, `current_stock`
       - `units_sold` (COUNT of matching sales), `revenue_eur` (SUM(item_price) / 100.0, rounded to 2 decimals)
       - `sell_through_pct` (units_sold / (capacity * p_days / 7.0) * 100, max 100, rounded to 1 decimal — weekly capacity-normalized throughput)
       - `avg_daily_units` (units_sold / p_days, rounded to 2 decimals)
       - `days_until_empty` (current_stock / NULLIF(avg_daily_units, 0), rounded to 0 decimals, null if avg_daily_units = 0)
       - `is_dead_stock` (true if units_sold = 0 AND capacity > 0)

    4. **Compute summary**: total_revenue_eur, total_units, avg_daily_revenue_eur (total_revenue / p_days)

    5. **Fetch paxcounter**: Latest `count` from `paxcounter` where `machine_id = p_machine_id`. Compute `conversion_rate` as `total_units / NULLIF(latest_count, 0)` rounded to 4 decimals, null if no paxcounter row.

    6. **Fetch refill history**: Last 5 `warehouse_transactions` rows where `transaction_type = 'outgoing_refill' AND reference_id = p_machine_id`, ordered by `created_at DESC`. Return array of `{ date (date part only), product_name, quantity_change (absolute value) }`.

    7. **Return JSON** assembled with `json_build_object(...)`:
    ```json
    {
      "machine": { "id": "...", "name": "...", "status": "..." },
      "period_days": 30,
      "summary": {
        "total_revenue_eur": 142.50,
        "total_units": 95,
        "avg_daily_revenue_eur": 4.75
      },
      "trays": [
        {
          "item_number": 1,
          "product_name": "Cola 0.5L",
          "capacity": 12,
          "current_stock": 3,
          "units_sold": 28,
          "revenue_eur": 39.20,
          "sell_through_pct": 83.3,
          "avg_daily_units": 0.93,
          "days_until_empty": 3,
          "is_dead_stock": false
        }
      ],
      "paxcounter": {
        "latest_count": 1240,
        "conversion_rate": 0.0766
      },
      "refill_history": [
        { "date": "2026-03-10", "product_name": "Cola 0.5L", "quantity_change": 10 }
      ]
    }
    ```

    IMPORTANT — Avoid these pitfalls:
    - Do NOT use RLS in this function — it uses `security definer` and validates company ownership manually via the `p_company_id` parameter. This allows it to be called from the edge function using the service role client.
    - Use `NULLIF(x, 0)` wherever division by zero is possible.
    - The `item_price` column in `sales` is stored in cents (integer), divide by 100.0 for EUR.
    - `products` join is LEFT JOIN from `machine_trays` — trays without a product get 'Empty Slot'.
    - Do NOT drop/alter any existing tables or functions.
    - The function must be idempotent (`create or replace function`).
    - Add a comment at the top: `-- AI Insights KPI aggregation RPC. Called by machine-insights edge function.`
    - Grant execute to `authenticated` and `service_role` at the end of the file.
