## Architecture
1. Customer taps product card → product selected state
2. Page calls POST /functions/v1/create-payment-intent with { subdomain, product_id, slot }
3. Response: { clientSecret, publishableKey, amount, currency }
4. Load Stripe.js dynamically, create Payment Element, mount in payment modal
5. Customer pays (Apple Pay / Google Pay / card)
6. On success: call POST /functions/v1/confirm-payment with { payment_intent_id, subdomain }
7. Show confirmation: "Credit delivered — select your product on the machine"
8. MDB note: customer must physically select product on machine display after paying

    **Extend the public-machine-data edge function response** — the public page needs to know if Stripe is configured. Add a `payment_enabled` boolean to the edge function response. In `public-machine-data/index.ts`, after fetching the company, check if `stripe_publishable_key` is non-null and add `payment_enabled: !!company.stripe_publishable_key` to the response. This avoids exposing any keys in the public data endpoint.

    **Extend `/m/[subdomain].vue`** with payment flow:

    1. **Stripe.js dynamic loading**:
       - On mount, if `data.payment_enabled`, dynamically load Stripe.js:
         ```typescript
         const stripePromise = ref<Promise<any> | null>(null)
         if (data.value?.payment_enabled) {
           stripePromise.value = new Promise((resolve) => {
             const script = document.createElement('script')
             script.src = 'https://js.stripe.com/v3/'
             script.onload = () => resolve((window as any).Stripe)
             document.head.appendChild(script)
           })
         }
         ```

    2. **Product card "Buy" button**:
       - Show a "Kaufen" / "Buy" button on each available product card (stock > 0) ONLY if `payment_enabled` is true
       - On tap: set `selectedProduct` ref → open payment modal
       - Style: prominent button below the stock bar, primary color

    3. **Payment modal** (bottom sheet like notify/wish modals):
       - Header: product name + price
       - Loading state: "Preparing payment..." while creating PaymentIntent
       - On open:
         a. POST to `/functions/v1/create-payment-intent` with `{ subdomain, product_id: selectedProduct.id, slot: selectedProduct.slot }`
         b. On response: init Stripe with `publishableKey`, create Elements with `clientSecret`, mount Payment Element
       - Payment Element container: `<div id="payment-element"></div>` inside the modal
       - "Pay {price}" button below the Payment Element
       - On submit:
         a. Call `stripe.confirmPayment({ elements, confirmParams: { return_url: window.location.href }, redirect: 'if_required' })`
         b. `redirect: 'if_required'` keeps the user on the page for most payment methods (Apple Pay, Google Pay, cards without 3DS)
         c. On success (paymentIntent.status === 'succeeded'):
            - Call POST `/functions/v1/confirm-payment` with `{ payment_intent_id, subdomain }`
            - Transition to confirmation state
         d. On error: show error message inline

    4. **Confirmation state** (replaces payment form in modal):
       - Green checkmark icon
       - "Payment successful!" / "Zahlung erfolgreich!"
       - "Credit has been sent to the machine. Please select your product on the machine display."
       - "Dismiss" button to close modal

    5. **Error handling**:
       - PaymentIntent creation fails (503 no Stripe, 409 out of stock): show error, allow dismiss
       - Stripe.js load fails: show error
       - Payment declined: show Stripe error message, allow retry
       - confirm-payment fails: show "Payment succeeded but credit delivery failed. Contact operator." with a warning — the webhook will deliver credit as backup

    6. **State management** (all page-local refs):
       ```typescript
       const selectedProduct = ref<Product | null>(null)
       const paymentLoading = ref(false)
       const paymentError = ref('')
       const paymentSuccess = ref(false)
       const confirmLoading = ref(false)
       ```

    - EN: buy, preparingPayment, pay, payAmount, paymentSuccess, paymentSuccessDesc, paymentFailed, paymentNotAvailable, outOfStock, creditDelivered, creditDeliveryFailed, dismiss, selectOnMachine
    - DE: kaufen, Zahlung wird vorbereitet, Bezahlen, {amount} bezahlen, Zahlung erfolgreich!, Ihr Guthaben wurde an den Automaten gesendet. Bitte wählen Sie Ihr Produkt am Automaten-Display., Zahlung fehlgeschlagen, Zahlung nicht verfügbar, Ausverkauft, Guthaben gesendet, Guthabenübertragung fehlgeschlagen. Kontaktieren Sie den Betreiber., Schließen, Produkt am Automaten wählen
