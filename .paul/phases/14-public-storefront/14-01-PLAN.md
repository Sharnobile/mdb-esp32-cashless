<objective>
## Goal
Create a public-facing storefront page for each vending machine, accessible via `/m/[subdomain]` URL without authentication. Displays machine info, products grouped by category with stock/price/availability. Allows visitors to subscribe for restock notifications on sold-out products and submit product wishes to the operator.

## Purpose
Allows end customers (people at the vending machine) to see what products are available, request notifications for out-of-stock items, and communicate product wishes to the operator — directly from their smartphone. Foundation for mobile payment (Phase 15).

## Output
- DB migration: `restock_subscriptions` and `product_wishes` tables
- Edge functions: `public-machine-data`, `subscribe-restock`, `submit-product-wish`
- Public Nuxt page `/m/[subdomain]` with mobile-first responsive design
- i18n translations (en/de)
</objective>

## Architecture Notes
- `embeddeds.subdomain` is a bigserial (auto-increment integer), unique per device — used as public URL identifier
- Lookup path: embeddeds(subdomain) → vendingMachine(embedded FK) → machine_trays(machine_id) → products(product_id) → product_category
- Current RLS: all tables require `authenticated` role — public page MUST use service_role via edge function
- Product images: public bucket `product-images`, URL pattern `{supabaseUrl}/storage/v1/object/public/product-images/{image_path}`
- Nuxt proxy: `routeRules['/functions/**']` proxies to Supabase — works for both SSR and CSR
- The `supabase-url.client.ts` plugin rewrites Supabase URLs client-side; public page avoids this by calling edge function via Nuxt proxy, not via Supabase client
- Existing multi-tenancy pattern: `company_id` column + RLS via `my_company_id()` for operator-facing data

    **Migration `20260409000000_public_storefront.sql`**:

    1. Create `restock_subscriptions` table:
       ```sql
       create table public.restock_subscriptions (
         id uuid not null default gen_random_uuid() primary key,
         created_at timestamptz not null default now(),
         machine_id uuid not null references public."vendingMachine"(id) on delete cascade,
         product_id uuid not null references public.products(id) on delete cascade,
         email text not null,
         notified_at timestamptz,  -- NULL = pending, set when notification sent
         company_id uuid not null references public.companies(id) on delete cascade
       );
       ```
       - Unique constraint on (machine_id, product_id, email) to prevent duplicates
       - Index on (machine_id, product_id, notified_at) for efficient lookup on restock
       - RLS enabled:
         - No anon policies (edge functions use service_role)
         - Authenticated SELECT: `company_id = my_company_id()` (operators can view their subscriptions)
         - Authenticated DELETE: `company_id = my_company_id() AND i_am_admin()` (operators can clean up)

    2. Create `product_wishes` table:
       ```sql
       create table public.product_wishes (
         id uuid not null default gen_random_uuid() primary key,
         created_at timestamptz not null default now(),
         machine_id uuid not null references public."vendingMachine"(id) on delete cascade,
         company_id uuid not null references public.companies(id) on delete cascade,
         wish_text text not null,
         email text,  -- optional contact
         status text not null default 'new' check (status in ('new', 'reviewed', 'dismissed'))
       );
       ```
       - RLS enabled:
         - Authenticated SELECT: `company_id = my_company_id()`
         - Authenticated UPDATE: `company_id = my_company_id() AND i_am_admin()` (status changes)
         - Authenticated DELETE: `company_id = my_company_id() AND i_am_admin()`

    **Edge function `public-machine-data`**:

    deno.json: `{ "imports": { "@supabase/supabase-js": "https://esm.sh/@supabase/supabase-js@2" } }`

    index.ts:
    - Accept GET with query param `subdomain` (required, integer)
    - Create Supabase client with `SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY` (bypasses RLS)
    - Lookup chain:
      1. `embeddeds` WHERE subdomain = param → get id, status, status_at
      2. `vendingMachine` WHERE embedded = embeddeds.id → get id, name, location_lat, location_lon
      3. `machine_trays` WHERE machine_id = machine.id AND product_id IS NOT NULL, joined with `products(id, name, sellprice, image_path, discontinued, product_category(name))`, ordered by item_number
    - Filter out discontinued products
    - Group trays by category name (null category → "Sonstige")
    - Return sanitized JSON (expose product_id for restock subscription, but NOT company/device IDs):
      ```json
      {
        "machine": { "name": "...", "location_lat": ..., "location_lon": ... },
        "machine_id": "uuid",
        "status": "online|offline",
        "status_at": "ISO timestamp",
        "categories": [
          {
            "name": "Getränke",
            "products": [
              { "id": "product-uuid", "name": "Coca Cola", "slot": 1, "price": 2.50,
                "stock": 8, "capacity": 18, "image_path": "abc.jpg", "available": true }
            ]
          }
        ]
      }
      ```
    - Note: expose `machine_id` and product `id` — needed for restock subscription and wish submission. These are UUIDs and not sensitive.
    - Error responses: 400 (missing subdomain), 404 (not found)
    - CORS headers: `Access-Control-Allow-Origin: *`, handle OPTIONS preflight

    **Edge function `subscribe-restock`**:

    deno.json: same import map.

    index.ts:
    - Accept POST with JSON body: `{ machine_id, product_id, email }`
    - Validate email format (basic regex)
    - Use service_role client to:
      1. Verify machine_id exists in vendingMachine → get company_id (via `company` FK)
      2. Verify product_id exists in products
      3. Insert into restock_subscriptions with company_id, using ON CONFLICT (machine_id, product_id, email) DO NOTHING
    - Return 200 `{ success: true }` or 409 if already subscribed
    - CORS headers

    **Edge function `submit-product-wish`**:

    deno.json: same import map.

    index.ts:
    - Accept POST with JSON body: `{ machine_id, wish_text, email? }`
    - Validate: wish_text required, max 500 chars
    - Use service_role client to:
      1. Verify machine_id exists in vendingMachine → get company_id
      2. Insert into product_wishes with company_id
    - Return 200 `{ success: true }`
    - Rate limit: simple check — max 10 wishes per machine per hour (count recent wishes, reject if over limit)
    - CORS headers

    **config.toml**: Add 3 function entries (follow existing pattern):
    ```toml
    [functions.public-machine-data]
    enabled = true
    verify_jwt = false
    import_map = "./functions/public-machine-data/deno.json"

    [functions.subscribe-restock]
    enabled = true
    verify_jwt = false
    import_map = "./functions/subscribe-restock/deno.json"

    [functions.submit-product-wish]
    enabled = true
    verify_jwt = false
    import_map = "./functions/submit-product-wish/deno.json"
    ```
