<objective>
## Goal
Create a database function to backfill historical sales with tax data, expose it in the frontend, and add validation warnings on pages that depend on tax configuration being complete.

## Purpose
Historical sales have NULL tax columns because tax infrastructure didn't exist when they were recorded. Users need a way to backfill them before exports (Phase 11) can produce complete data. Validation ensures users configure tax classes on all categories before attempting exports.

## Output
- DB function `backfill_sales_tax()` that stamps historical sales based on current tax config
- Backfill button in Tax Settings section on Settings page
- Validation helper that checks for unconfigured categories
- Warning banner on Products page when categories lack tax classes
</objective>

  <action>
    **DB Migration — backfill_sales_tax RPC:**
    Create `backfill_sales_tax(p_company_id uuid)` as SECURITY DEFINER function:
    - UPDATE sales s SET
        tax_rate_snapshot = tr.rate,
        price_net = ROUND(s.item_price / (1 + tr.rate), 4),
        tax_amount = ROUND(s.item_price - ROUND(s.item_price / (1 + tr.rate), 4), 4)
      FROM vendingMachine vm, machine_trays mt, products p, product_category pc, tax_rates tr, companies c
      WHERE s.tax_rate_snapshot IS NULL
        AND s.item_price IS NOT NULL
        AND s.machine_id = vm.id
        AND vm.company = p_company_id
        AND mt.machine_id = vm.id AND mt.item_number = s.item_number
        AND p.id = mt.product_id
        AND pc.id = p.category
        AND COALESCE(p.tax_class_id, pc.tax_class_id) = tr.tax_class_id
        AND tr.company_id = p_company_id
        AND tr.country_code = COALESCE(vm.country_code, c.country_code, 'DE')
        AND c.id = p_company_id
        AND tr.valid_from <= s.created_at::date
        AND (tr.valid_to IS NULL OR tr.valid_to >= s.created_at::date)
    - Use a subquery or lateral join to pick the newest valid rate (ORDER BY valid_from DESC LIMIT 1) when multiple rates match
    - RETURNS integer (count of updated rows via GET DIAGNOSTICS)
    - Must verify p_company_id matches caller's company (use my_company_id() check or manual auth)
