<objective>
## Goal
Enhance AI insights with richer data, caching, new recommendation types, trend comparison, and eliminate misleading current-stock warnings.

## Purpose
Current insights are limited to single-period KPIs and treat momentary stock levels as critical. Operators need trend-aware, actionable recommendations that understand warehouse context and don't flag easily-resolved stock snapshots.

## Output
- Updated RPC with warehouse stock, trend data (current vs previous period), and no current_stock emphasis
- New `machine_insights_cache` table for server-side caching
- Updated edge function with caching logic, improved prompt, new recommendation types
- Updated frontend with new type labels, cache freshness indicator, and trend display
</objective>

## AC-1: RPC includes warehouse stock and trend comparison
```gherkin

And the prompt explicitly instructs Claude to ignore current_stock as it's a transient snapshot
```

<task type="auto">
  <name>Task 1: Update RPC and add cache table</name>
  <files>Docker/supabase/migrations/20260319100000_enhanced_insights.sql</files>
  <action>
    Create a new migration that:

    1. **Creates `machine_insights_cache` table:**
       - `id` uuid PK default gen_random_uuid()
       - `machine_id` uuid NOT NULL references "vendingMachine"(id) ON DELETE CASCADE
       - `company_id` uuid NOT NULL references companies(id) ON DELETE CASCADE
       - `period_days` int NOT NULL
       - `response` jsonb NOT NULL (the full AI response)
       - `created_at` timestamptz NOT NULL default now()
       - UNIQUE constraint on (machine_id, period_days)
       - RLS enabled, policy: select/insert/update/delete for service_role only (edge function uses service role)

    2. **Creates `get_machine_insights_kpis_v2` (new function, keep v1 for backward compat):**
       - Same signature as v1: `(p_machine_id uuid, p_company_id uuid, p_days int default 30) returns json`
       - Same company ownership validation
       - **Remove** `current_stock` and `days_until_empty` from per-tray output
       - **Add** `warehouse_stock` section: query `warehouse_stock_batches` joined with `warehouses` and `products` for all products assigned to this machine's trays, grouped by product. Return: `[{ product_name, warehouse_name, available_qty }]` where `available_qty = SUM(quantity)` of non-expired batches (expiry_date IS NULL OR expiry_date > now())
       - **Add** `trends` section: run the same revenue/units aggregation for the PREVIOUS period (v_window_start - p_days days to v_window_start) and compute:
         - `prev_revenue_eur`, `prev_total_units`
         - `revenue_change_pct`: ((current - prev) / prev * 100), null if prev is 0
         - `units_change_pct`: same logic
       - **Keep** all existing fields except current_stock and days_until_empty

    Important:
    - Use `security definer` + manual company check (same pattern as v1)
    - Use `round(..., 2)` with explicit `::numeric` casts (PostgreSQL needs this for float→numeric)
    - Grant execute to authenticated and service_role
    - Do NOT drop `get_machine_insights_kpis` v1 — keep it for backward compatibility
  </action>
  <verify>Migration SQL is syntactically valid. Cache table has correct constraints. V2 function returns warehouse_stock and trends.</verify>
  <done>AC-1 and AC-4 (data layer) satisfied: RPC returns warehouse stock, trends, no current_stock</done>
</task>

<task type="auto">
  <name>Task 2: Update edge function with caching, new prompt, new types</name>
  <files>Docker/supabase/functions/machine-insights/index.ts</files>
  <action>
    Update the edge function:

    1. **Caching logic (before calling Anthropic):**
       - After auth + body parsing, query `machine_insights_cache` for matching (machine_id, period_days)
       - If found and `created_at` is less than 6 hours ago, return the cached `response` with `"cached": true` added
       - If stale or missing, proceed to generate new insights

    2. **Switch to v2 RPC:**
       - Change RPC call from `get_machine_insights_kpis` to `get_machine_insights_kpis_v2`

    3. **Updated prompt — key changes:**
       - Add warehouse_stock data to the MACHINE DATA section
       - Add trends data (period comparison)
       - Add explicit instruction: "IMPORTANT: Do NOT recommend urgent restocking or flag low current machine stock. Current tray stock is a transient snapshot that changes with each refill visit. Focus on sell-through rates, trends, and product performance instead."
       - Add new recommendation types to the allowed list:
         - `pricing_strategy`: for products where conversion is low but foot traffic is high, or where price-per-unit is outlier
         - `cross_selling`: for product combinations that could work well together based on sales patterns
       - Update rules section with new type guidance
       - Remove rules referencing `days_until_empty` and `current_stock`
       - Include trend data interpretation: "Use the trends section to highlight improving or declining performance compared to the previous period."

    4. **Increase max_tokens to 2048** (more recommendation types = longer responses)

    5. **Cache write (after successful Anthropic response):**
       - Upsert into `machine_insights_cache` with the full response JSON
       - Use ON CONFLICT (machine_id, period_days) DO UPDATE

    6. **Add `force_refresh` request parameter:**
       - If `force_refresh: true` in body, skip cache read (but still write cache after)
       - Default: false

    7. **Include `cached: false` in fresh responses**
  </action>
  <verify>Edge function handles cache hit (returns cached + no API call), cache miss (calls API + writes cache), and force_refresh. New prompt includes warehouse stock, trends, and stock-snapshot disclaimer.</verify>
  <done>AC-2 (caching), AC-3 (new types in prompt), AC-4 (no stock warnings) satisfied</done>
</task>
