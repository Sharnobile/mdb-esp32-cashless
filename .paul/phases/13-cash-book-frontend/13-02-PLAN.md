<objective>
## Goal
Add entry recording dialogs (withdrawal with Soll/Ist comparison, correction, payout, reversal/Storno) and PDF export to the Kassenbuch page. This completes the full cash book feature.

## Purpose
Operators need to record cash events at machines: withdraw cash (comparing counted vs theoretical amount), correct mistakes (via Storno + new entry), pay out to bank account, and export the cash book as a GoBD-compliant PDF for their accountant.

## Output
- Withdrawal dialog with theoretical/counted comparison + difference display
- Correction dialog for adjustments
- Payout dialog for bank transfers
- Reversal (Storno) functionality per entry
- PDF export (client-side generation)
</objective>

## AC-1: Withdrawal dialog with Soll/Ist
```gherkin
Given a Barkasse is selected with theoretical cash calculated
When the user clicks "Entnahme erfassen"
Then a dialog opens showing:
  - Theoretical amount pre-filled (read-only, from get_theoretical_cash)
  - Per-machine breakdown (which machine has how much theoretical cash)
  - "Gezählter Betrag" input field (what was actually counted)
  - Auto-calculated difference (counted - theoretical)
  - If difference != 0: warning with "Differenz: X.XX €" in amber
  - Description text field (pre-filled "Geldentnahme - Bankeinzahlung")
  - Machine selector (optional, which machine was cash taken from)
And on submit, creates a withdrawal entry with counted_amount and expected_amount
And if difference exists, shows option to auto-create a correction entry for the difference
```

## AC-2: Correction dialog
```gherkin
Given a Barkasse is selected
When the user clicks "Korrektur erfassen"
Then a dialog opens with:
  - Amount input (positive or negative)
  - Description text field
And on submit, creates a correction entry
And the balance updates accordingly
```

## AC-3: Payout dialog
```gherkin
Given a Barkasse with positive balance
When the user clicks "Auszahlung erfassen"
Then a dialog opens with:
  - Amount input (max = current balance)
  - Description text field (pre-filled "Auszahlung auf Bankkonto")
And on submit, creates a payout entry (negative amount)
```

## AC-4: Reversal (Storno)
```gherkin
Given an entry that is not already reversed
When the user clicks the "Stornieren" button on that entry row
Then a confirmation dialog appears: "Eintrag #X stornieren?"
And on confirm, creates a reversal entry via the composable
And the original entry shows as "Storniert" (dimmed, strikethrough)
And already-reversed entries have no Storno button
```

## AC-5: PDF export
```gherkin
Given a Barkasse with entries
When the user clicks "Als PDF exportieren"
Then a PDF is generated client-side containing:
  - Barkasse name + company info
  - KPI summary (current balance, withdrawals, corrections)
  - Full entry table (all entries, not just filtered)
  - GoBD compliance note + activation date + entry count
  - Hash of last entry for verification
And the PDF downloads automatically
```
