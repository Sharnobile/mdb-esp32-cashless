<objective>
## Goal
Build the frontend UI for managing tax classes, tax rates, and country assignments. Users can configure tax settings, assign tax classes to product categories, and set country codes on companies and machines. On organization creation, automatically seed tax classes and rates from system reference rates.

## Purpose
Without UI, the DB infrastructure from Plan 01 is unusable. Users need to configure their tax classes, assign them to categories, and set their country — all before tax exports (Phase 11) can work.

## Output
- New composable: `useTaxSettings` for tax class + rate CRUD
- Tax settings section on Settings page (admin-only)
- Tax class dropdown on category form (Products page)
- Country selector on machine detail page
- Country selector on Settings page (company-level)
- Auto-seeding of tax classes + rates on organization creation
- Full i18n (EN + DE)
</objective>

<acceptance_criteria>

## AC-1: Tax settings UI on Settings page
```gherkin
Given the user is an admin
When they navigate to Settings
Then they see a "Tax Settings" section with:
  - Company country selector (dropdown of common countries: DE, AT, CH, FR, etc.)
  - List of tax classes with name + current rate for company country
  - Ability to add/edit/delete tax classes
  - Ability to add/edit tax rates per class per country
  - "Seed from defaults" button that copies system_tax_rates for selected country
```

## AC-2: Tax class assignment on product categories
```gherkin
Given tax classes exist for the company
When the user creates or edits a product category on the Products page
Then the category form includes a required "Tax Class" dropdown
And the dropdown shows all company tax classes (e.g. "Standard (19%)", "Reduced (7%)")
And the selected tax class is saved to product_category.tax_class_id
```

## AC-3: Country selector on machines
```gherkin
Given a machine exists
When the user views the machine detail page
Then there is a country field showing the machine's country
And it defaults to "(Company default: DE)" when NULL
And the user can override it to a different country
```

## AC-4: Auto-seed on organization creation
```gherkin
Given a new user creates an organization
When the organization is created successfully
Then tax classes "standard" and "reduced" are automatically created
And DE tax rates (19%, 7%) are automatically seeded for those classes
And the company country_code is set to 'DE'
```

## AC-5: i18n support
```gherkin
Given the app is set to German or English
When the user views any tax-related UI
Then all labels, placeholders, and messages are translated
```

<task type="auto">
  <name>Task 1: Create useTaxSettings composable and update create-organization</name>
  <files>
    management-frontend/app/composables/useTaxSettings.ts,

    management-frontend/i18n/locales/de.json
  </files>
  <action>
    **useTaxSettings.ts:**
    Create composable exposing:
    - `taxClasses: Ref<TaxClass[]>` — fetched from tax_classes WHERE company_id
    - `taxRates: Ref<TaxRate[]>` — fetched from tax_rates WHERE company_id
    - `systemRates: Ref<SystemTaxRate[]>` — fetched from system_tax_rates (read-only)
    - `companyCountry: Ref<string>` — from companies.country_code
    - `fetchTaxClasses()` — SELECT from tax_classes
    - `fetchTaxRates()` — SELECT from tax_rates with join to tax_classes for name
    - `fetchSystemRates()` — SELECT from system_tax_rates
    - `createTaxClass(name, description?)` — INSERT into tax_classes
    - `updateTaxClass(id, name, description?)` — UPDATE tax_classes
    - `deleteTaxClass(id)` — DELETE from tax_classes (CASCADE deletes rates)
    - `createTaxRate(taxClassId, countryCode, rate, name, validFrom)` — INSERT into tax_rates
    - `updateTaxRate(id, updates)` — UPDATE tax_rates
    - `deleteTaxRate(id)` — DELETE from tax_rates
    - `seedFromSystem(countryCode)` — Read system_tax_rates for country, create matching tax_classes (if not exist) and tax_rates for company
    - `updateCompanyCountry(countryCode)` — UPDATE companies SET country_code

    Types:
    ```typescript
    interface TaxClass { id: string; name: string; description: string | null; sort_order: number }
    interface TaxRate { id: string; tax_class_id: string; tax_class_name?: string; country_code: string; rate: number; name: string; valid_from: string; valid_to: string | null; is_inclusive: boolean }
    interface SystemTaxRate { id: number; country_code: string; tax_class_name: string; rate: number; name: string; valid_from: string }
    ```

    **create-organization/index.ts:**
    After creating the company and org member, add:
    1. INSERT tax_classes: "standard" (sort_order 0) and "reduced" (sort_order 1) for new company_id
    2. INSERT tax_rates: standard→DE→0.19 and reduced→DE→0.07 with valid_from '2007-01-01'
    3. UPDATE companies SET country_code = 'DE' WHERE id = company_id
    Use adminClient (service role) for these inserts since user RLS may not be active yet.

    **i18n:** Add tax section keys to both en.json and de.json:
    - tax.title, tax.taxClasses, tax.taxRates, tax.addTaxClass, tax.editTaxClass
    - tax.className, tax.rate, tax.country, tax.validFrom, tax.validTo
    - tax.seedFromDefaults, tax.seedSuccess, tax.companyCountry
    - tax.standard, tax.reduced, tax.exempt, tax.inclusive
    - tax.noTaxClasses, tax.noTaxRates, tax.deleteTaxClassConfirm
    - tax.assignTaxClass, tax.inheritFromCategory, tax.companyDefault
    - products.taxClass (for category form)
    - machines.country, machines.countryInherited
  </action>
  <verify>
    - useTaxSettings.ts exports all listed functions and types
    - create-organization/index.ts seeds tax data after company creation
    - i18n keys exist in both en.json and de.json
  </verify>
  <done>AC-4, AC-5 partially satisfied</done>
