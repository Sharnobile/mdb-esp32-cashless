<objective>
## Goal
Create the complete database infrastructure for tax-compliant sales recording: tax classes, tax rates, system reference rates, country codes on companies/machines, tax columns on sales, and a stamp trigger that automatically records the applicable tax rate at sale time.

## Purpose
Foundation for tax-compliant exports (DATEV, CSV). Without this DB layer, no tax reporting is possible. Must be backward-compatible — existing data and firmware must continue to work unchanged.

## Output
- Two new tables: `tax_classes`, `tax_rates`
- One new read-only table: `system_tax_rates` (EU reference rates)
- New columns on `product_category`, `products`, `companies`, `vendingMachine`, `sales`
- Updated `stamp_machine_and_decrement_stock` trigger with tax stamping
- RLS policies for all new tables/columns
- Seeded DE + AT standard rates
</objective>

<task type="auto">
  <name>Task 1: Create tax infrastructure migration</name>
  <files>Docker/supabase/migrations/20260406000000_tax_infrastructure.sql</files>
  <action>
    Create migration with:

    1. **tax_classes table:**
       - id (uuid PK), company_id (uuid FK companies NOT NULL), name (text NOT NULL), description (text), sort_order (int DEFAULT 0), created_at (timestamptz DEFAULT now())
       - UNIQUE(company_id, name)
       - RLS: SELECT/INSERT/UPDATE/DELETE for authenticated where company_id = my_company_id()

    2. **tax_rates table:**
       - id (uuid PK), company_id (uuid FK companies NOT NULL), tax_class_id (uuid FK tax_classes NOT NULL ON DELETE CASCADE), country_code (CHAR(2) NOT NULL DEFAULT 'DE'), rate (NUMERIC(6,4) NOT NULL), name (text NOT NULL), valid_from (DATE NOT NULL DEFAULT CURRENT_DATE), valid_to (DATE), is_inclusive (BOOLEAN NOT NULL DEFAULT TRUE), created_at (timestamptz DEFAULT now())
       - UNIQUE(company_id, tax_class_id, country_code, valid_from)
       - RLS: SELECT/INSERT/UPDATE/DELETE for authenticated where company_id = my_company_id()

    3. **system_tax_rates table** (reference, not per-company):
       - id (serial PK), country_code (CHAR(2) NOT NULL), tax_class_name (text NOT NULL), rate (NUMERIC(6,4) NOT NULL), name (text NOT NULL), valid_from (DATE NOT NULL), valid_to (DATE)
       - RLS: SELECT only for authenticated (read-only reference)
       - No INSERT/UPDATE/DELETE policies

    4. **Alter existing tables:**
       - product_category: ADD COLUMN tax_class_id uuid REFERENCES tax_classes(id) ON DELETE SET NULL
       - products: ADD COLUMN tax_class_id uuid REFERENCES tax_classes(id) ON DELETE SET NULL
       - companies: ADD COLUMN country_code CHAR(2) DEFAULT 'DE'
       - vendingMachine: ADD COLUMN country_code CHAR(2)
       - sales: ADD COLUMN tax_rate_snapshot NUMERIC(6,4), ADD COLUMN tax_amount NUMERIC(10,4), ADD COLUMN price_net NUMERIC(10,4)

    5. **Update stamp_machine_and_decrement_stock trigger:**
       After the existing stock decrement logic, add tax stamping:
       - Resolve country: COALESCE(vendingMachine.country_code, companies.country_code, 'DE')
       - Resolve tax_class: Look up product via machine_trays → products → COALESCE(products.tax_class_id, product_category.tax_class_id)
       - If both country and tax_class found: query tax_rates for matching (company_id, tax_class_id, country_code) where valid_from <= NEW.created_at AND (valid_to IS NULL OR valid_to >= NEW.created_at), ORDER BY valid_from DESC LIMIT 1
       - If rate found: set NEW.tax_rate_snapshot, calculate NEW.price_net = ROUND(NEW.item_price / (1 + rate), 4), NEW.tax_amount = ROUND(NEW.item_price - NEW.price_net, 4)
       - If not found: leave all three NULL (graceful fallback)
       - Important: The trigger must NEVER block a sale due to missing tax config

    Avoid:
    - NOT NULL on new columns of existing tables (breaks backward compat)
    - Blocking sales if tax config is missing
    - Using float8 for tax columns (use NUMERIC for precision)
    - Dropping or renaming existing columns
  </action>
